# Security Policy

## Supported Versions

The security team actively monitors and provides security updates for the following versions of the IEEESoc Bot Automation Engine:

| Version | Supported |
| :--- | :--- |
| 2.x | :white_check_mark: Yes |
| 1.x | :x: No |

## Reporting a Vulnerability

We take the security of our fellowship platform, scoring engine, and anti-cheat checks very seriously. If you discover a vulnerability or a loophole that allows bypassing points constraints (e.g., signature verification bypasses, points inflation, or cheating the self-merge detection), please report it responsibly.

**Please do not report security vulnerabilities via public GitHub issues.**

Instead, please report security concerns directly to the IEEE Core Committee at:
📩 **ieee-soc-security@gehu.ac.in**

When submitting a report, please include:
1.  A detailed description of the vulnerability or scoring bypass.
2.  Step-by-step instructions (with proof-of-concept payloads or script steps) to reproduce the issue.
3.  The impact of the issue on the leaderboard integrity.

Once a report is received:
*   We will acknowledge receipt of your vulnerability report within 48 hours.
*   We will work on a fix and coordinate a release date with you.
*   We request that you keep the report confidential until the fix is deployed and publicly released.
